# CampusON 사용자 매뉴얼

## 학생 가이드

### 1. 회원가입
1. `/register` 페이지에서 이름, 이메일, 비밀번호, 학과, 학번 입력
2. 비밀번호: 영문 + 숫자 8자 이상
3. 학번: 8~10자리 숫자
4. 가입 즉시 로그인 상태로 대시보드 이동

### 2. 진단 테스트 (최초 1회)
1. 대시보드에서 "진단 테스트 시작하기" 클릭
2. 학과별 30문항 자동 출제 (난이도 혼합)
3. 문항별 선택지 클릭 후 "다음"
4. 모든 문항 응답 후 "제출"
5. 결과 확인: 전체 점수, 과목별 정답률, 학습 수준
6. AI 프로파일 자동 생성 (취약영역, 추천 학습 경로)

### 3. 문제 풀이
1. 대시보드 → "문제 풀이" 클릭
2. AI가 추천한 맞춤형 문제셋 자동 제공
3. 선택지 클릭 → "제출" → 즉시 채점
4. 정답/오답 확인 + 공식 해설
5. "AI 해설 보기" → RAG 기반 상세 해설 (인용 포함)
6. "다음 문제"로 계속 학습

### 4. AI 튜터 채팅
1. 대시보드 → "AI 튜터" 클릭
2. 자유롭게 질문 입력
3. AI가 학생의 학과, 수준, 취약영역을 고려하여 답변
4. 지식베이스(KB) 자료 인용 시 [숫자] 형태로 표시
5. 참고 자료 목록도 함께 확인 가능

### 5. 오답노트
1. 대시보드 → "오답노트" 클릭
2. 반복 오답이 많은 문제부터 우선 표시
3. 오답 유형: 개념 부족 / 헷갈림 / 실수 / 응용 부족
4. "다시 풀기"로 재학습
5. "해결된 문제도 표시" 체크박스로 전체 확인

### 6. 학습 리포트
1. 대시보드 → "학습 리포트" 클릭
2. 전체 정답률, 총 시도 횟수
3. 시계열 추이 (일간/주간/월간 선택)
4. 과목별 정답률 바 차트
5. 오답 유형 분포

## 교수 가이드

### 1. 학과 학생 관리
1. 대시보드 → "관리" 클릭
2. 본인 학과 학생 목록 확인
3. 학생 클릭 → 개별 학습 통계, 진단 결과, 취약영역 조회
4. 오답률 높은 학생 식별

### 2. AI 답변 품질 검토
- Swagger UI (`/docs`)에서 `/ai/logs` API로 호출 이력 확인
- 부정확한 답변 발견 시 개발팀에 피드백

### 3. KB 문서 검수
- `/api/v1/kb/documents` → 문서 목록 조회
- `POST /kb/documents/{id}/review` → 검수 승인/발행

## 관리자 가이드

### 1. 사용자 관리
- "관리" 페이지에서 전체 사용자 목록 확인
- 역할/상태별 필터링

### 2. 문제은행 관리
- CSV 일괄 업로드: `/api/v1/questions/bulk-upload`
- Swagger UI에서 직접 문제 CRUD

### 3. KB 지식베이스 관리
- 문서 적재: `POST /api/v1/kb/documents`
- 검수 워크플로우: DRAFT → REVIEWED → PUBLISHED
- 삭제: `DELETE /api/v1/kb/documents/{id}`

### 4. 시스템 모니터링
- `/api/v1/health` — 서버 상태
- `/api/v1/ai/logs` — AI 호출 이력 + 비용 추적
- `audit_logs` 테이블 — 전체 API 감사 로그

## FAQ

**Q: 진단 테스트를 다시 볼 수 있나요?**
A: 아니요. 진단 테스트는 학생당 1회만 가능합니다. 결과는 `/diagnostic/me`에서 확인 가능합니다.

**Q: AI 해설이 "MOCK"이라고 나와요.**
A: 개발 환경에서는 Mock LLM이 기본 활성화됩니다. 실제 Claude를 사용하려면 `.env`에서 `LLM_PROVIDER=anthropic`과 `ANTHROPIC_API_KEY`를 설정하세요.

**Q: 문제가 추천되지 않아요.**
A: 진단 테스트를 먼저 응시해야 AI 프로파일이 생성되고 추천이 활성화됩니다.

**Q: 비밀번호를 잊었어요.**
A: `POST /api/v1/auth/password-reset/request`에 이메일을 전송하면 재설정 토큰이 발급됩니다 (개발 환경에서는 응답에 토큰이 포함됩니다).
